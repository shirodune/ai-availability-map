## Data model

All countries keyed by **ISO 3166-1 numeric** code (matches `world-atlas` TopoJSON `feature.id`). Availability uses **exception semantics**: a country is *available* for a product+layer unless listed in that layer's `unavailable` set, or `unknown` if its source is stale. (The scraper computes `unavailable = world-atlas universe ‚àí supported-set`; matches the design's "default available, list exceptions" model.)

## Architecture

```
Schedule (GitHub Actions, ~daily)
  ‚Üí per-source adapters: {chatgpt,claude,gemini} √ó {app,api}  (fetch + parse)
  ‚Üí normalize country names ‚Üí ISO numeric (+ override table)
  ‚Üí validate (sanity bounds); on failure: keep last-known-good, set stale=true
  ‚Üí write availability.json  ‚Üí diff vs previous ‚Üí append changelog.json
  ‚Üí commit ‚Üí static deploy (GitHub Pages / Netlify / Vercel)
Runtime: React SPA fetches /data/*.json, merges notes, renders map/table/changelog.
```

A broken source can never blank the map: validation rejects an implausible scrape (0 or ~all countries) and the previous dataset is retained, with that product+layer surfaced as **stale/unknown**.

## View model

- **Layer:** App | API (toggle). Feature spotlights apply to the **App** layer.
- **View within a layer:** Combined (count 0‚Äì3) ¬∑ ChatGPT ¬∑ Claude ¬∑ Gemini ¬∑ Feature spotlight.
- **Page mode:** Map ¬∑ Table ¬∑ Changelog (header switch).

## Components (`src/`)

`main.jsx` ¬∑ `App.jsx` (state: theme, mode, layer, view, featureId, selectedId, query) ¬∑ `components/`: `Header` (titles + `StatChip`s), `Toolbar` (`LayerToggle`, product `Seg`, `FeatureSubtabs`, `Search`), `SpotlightBar`, `MapView` (@visx/geo), `Legend`, `DetailPanel` + `AvailRow` + `SourceLink`, `SideList`, `TableView`, `ChangelogView`, `Footer` (methodology + sources), `ThemeSwitcher`. `lib/`: `data.js` (load + derive status), `geo.js` (projections), `iso.js`.

## Map

`@visx/geo` `CustomProjection` driven by `d3-geo` (`geoEqualEarth` default, `geoNaturalEarth1`, `geoMercator`). Paths memoized; layer/view/theme changes only swap fills (`var(--c0..c3)`). Hover tooltip + click selection as in the prototype.

## Risks / notes

- **Scraper fragility** is the main risk ‚Äî provider pages aren't APIs and may be JS-rendered. Mitigation: per-source adapters, Step-0 reconnaissance of each page's shape, `cheerio` for static HTML (escalate to headless only if required), validation + last-known-good. Ship with a hand-seeded best-effort dataset so the app works before the first scrape.
- **API-layer data** is genuinely different from app data (e.g. OpenAI API vs ChatGPT app country lists); adapters fetch distinct source URLs per layer.
- Git is required (changelog history, CI). Repo initialized at scaffold; **no commits until the user asks**.
